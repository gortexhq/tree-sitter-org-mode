# tree-sitter-org-mode

Vendored fork of [zac-garby/tree-sitter-org-mode](https://github.com/zac-garby/tree-sitter-org-mode)
(MIT) — a tree-sitter parser for [Org](https://orgmode.org/worg/org-syntax.html)
documents — packaged for Go and consumed by Gortex.

The grammar source (`grammar.js`, `node-types.json`, `grammar.json`) and the
non-Go bindings are intentionally omitted. Only the artefacts needed at build
and run time are kept:

- `src/parser.c` — generated tree-sitter parser
- `src/scanner.c` — custom external scanner (Org is heavily context-sensitive)
- `src/tree_sitter/*.h` — tree-sitter parser headers
- `bindings/go/binding.go` — CGO entry point exposing `Language()`

## Use

```go
import (
    tsorg "github.com/gortexhq/tree-sitter-org-mode/bindings/go"
    "github.com/tree-sitter/go-tree-sitter"
)

lang := tree_sitter.NewLanguage(tsorg.Language())
```

The grammar's tree-sitter symbol is `tree_sitter_orgmode` (no underscore in
"orgmode"), matching upstream.

## Updating

To pull a newer upstream snapshot:

```sh
git clone --depth 1 https://github.com/zac-garby/tree-sitter-org-mode /tmp/upstream
cp /tmp/upstream/src/parser.c   src/
cp /tmp/upstream/src/scanner.c  src/
cp /tmp/upstream/src/tree_sitter/*.h src/tree_sitter/
```

No regeneration is required — the upstream `parser.c` is already generated by
`tree-sitter generate`.
